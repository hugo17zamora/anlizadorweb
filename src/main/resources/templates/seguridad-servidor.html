Seguridad del servidor

Analiza cabeceras de seguridad y configuraciones que protegen al usuario frente a ataques.

Aspectos analizados:

Content Security Policy (CSP)

Comprueba si el servidor implementa la cabecera de seguridad Content Security Policy.

CSP define desde qué ubicaciones puede cargar recursos el navegador. Su utilización se considera una buena práctica de seguridad, ya que ayuda a prevenir ataques como Cross-Site Scripting (XSS).

X-Frame-Options

Analiza si la página utiliza la cabecera X-Frame-Options.

Esta cabecera protege frente a ataques de clickjacking al controlar si una página puede cargarse dentro de un iframe. Los valores recomendados son DENY y SAMEORIGIN.

X-Content-Type-Options

Verifica la existencia de la cabecera X-Content-Type-Options.

Esta cabecera impide que el navegador intente adivinar el tipo de contenido de un archivo. El valor recomendado es nosniff, que obliga al navegador a respetar el tipo declarado por el servidor.

Referrer-Policy

Comprueba la política utilizada para compartir información de origen entre páginas web.

Configuraciones como strict-origin o no-referrer limitan la información enviada y ofrecen una mayor protección de la privacidad del usuario.

Permissions-Policy

Analiza la configuración de la cabecera Permissions-Policy.

Esta cabecera permite restringir o permitir el acceso a determinadas funcionalidades del navegador, como la cámara, el micrófono o la geolocalización, reduciendo así la superficie de ataque.
