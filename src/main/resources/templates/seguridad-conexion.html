Seguridad de conexión

Valora el cifrado, la integridad y la autenticidad de los datos intercambiados entre el servidor y el cliente.

Aspectos analizados:

Uso de HTTPS

Comprueba si el sitio web utiliza HTTPS para proteger las comunicaciones entre el navegador y el servidor.

HTTPS cifra la información intercambiada, evitando que terceros puedan interceptar o modificar datos sensibles como contraseñas o información personal. Sin HTTPS, la comunicación es vulnerable a diferentes ataques.

Validez del certificado digital

Verifica que el certificado SSL/TLS del sitio web se encuentre vigente.

El certificado permite comprobar la identidad del sitio web. Si está caducado o no es válido, disminuye la confianza en la página y pueden aumentar los riesgos de seguridad.

Tipo de firma del certificado

Analiza si el certificado ha sido emitido por una entidad reconocida o si se trata de un certificado autofirmado.

Los certificados autofirmados no ofrecen garantías suficientes sobre la identidad del sitio web, por lo que pueden representar un mayor riesgo para los usuarios.

Autoridad de Certificación (CA)

Comprueba qué Autoridad de Certificación ha emitido el certificado digital.

Las CA reconocidas verifican la identidad de los sitios web y proporcionan una cadena de confianza que permite garantizar la autenticidad del certificado.

Versión TLS utilizada

Analiza la versión del protocolo SSL/TLS utilizada por el servidor.

Las versiones SSL 2.0, SSL 3.0, TLS 1.0 y TLS 1.1 se consideran obsoletas debido a la existencia de vulnerabilidades conocidas. TLS 1.2 y TLS 1.3 ofrecen mecanismos criptográficos más seguros y son las versiones recomendadas actualmente.
